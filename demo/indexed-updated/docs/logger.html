Class: Logger UPDATED

📁 logger.py

Logger with timestamp support for operation tracking.

Constructor:

Logger(level: str = "INFO")

Initialize logger with given level.

Parameters: level ∈ {INFO, DEBUG, WARNING, ERROR}

Methods:

info(message: str)

Log info-level message with timestamp if level is INFO or DEBUG.

debug(message: str)

Log debug-level message with timestamp if level is DEBUG.

warning(message: str) ✨ NEW

Log warning-level message with timestamp.

Condition: Output if level is INFO, WARNING, or DEBUG

Destination: stderr (warning-level importance)

error(message: str)

Log error message with timestamp (always output).

Destination: stderr

Graph Relationships:

  • IMPORTS: config.Config
  • CALLED BY: calculator.Calculator methods
  • CALLED BY: main.main()

Changes Made in This Update:

  • ⏰ Added timestamp support via _format_message() helper
  • ✨ Added new warning() method for intermediate severity
  • 📝 All methods now output timestamps in format [YYYY-MM-DD HH:MM:SS]
  • 🧠 Improved separation of concerns with _format_message()

Output Format Examples:

[INFO] [2026-03-27 14:30:15] add(10, 5) = 15
[DEBUG] [2026-03-27 14:30:16] divide(20, 4) = 5.0
[WARNING] [2026-03-27 14:30:17] Division by zero attempted
[ERROR] [2026-03-27 14:30:18] Invalid input type

Usage Example:

from logger import Logger

logger = Logger("INFO")
logger.info("Operation started")      # Prints with timestamp
logger.debug("Debug info")              # Not printed (level=INFO)
logger.warning("Precision loss")       # Prints with timestamp
logger.error("An error occurred")      # Prints to stderr with timestamp
